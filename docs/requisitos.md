# Requisitos de software

## Requisitos funcionales

1. El sistema permite registrar incidencias de la vía pública.
2. El sistema permite clasificar la incidencia como BACHE, ALUMBRADO, BASURA, SEGURIDAD o EMERGENCIA.
3. El sistema permite adjuntar imagen, video o audio como evidencia.
4. El sistema permite listar las incidencias registradas.
5. El sistema permite cambiar el estado de una incidencia.
6. El sistema permite registrar usuarios básicos y hacer login simple.

## Requisitos no funcionales

1. El sistema debe ser sencillo de ejecutar en una computadora local.
2. El backend debe exponer una API REST.
3. Los datos deben guardarse en MongoDB.
4. El código debe estar separado por capas simples.
